# Solara Flutter 版设计（不含 Web）

日期：2026-03-12  
目标位置：/Users/aay/自有项目/solara_flutter

## 1. 目标与范围
- 目标：将 Solara 的完整功能迁移为 Flutter 应用，覆盖 Android、iOS、Windows、macOS、Linux（Web 暂不实现）。
- 后端：继续使用现有 Cloudflare Pages Functions（保持协议不变），后续可替换自有后端或自建服务。
- 交付：完整功能覆盖、可运行的多平台工程结构、可配置的 API 基地址。

## 2. 架构与落地形态
- 新建独立 Flutter 项目（不改现有仓库移动端）。
- 分层结构：
  - presentation：页面与组件（播放器、搜索、列表、歌词、收藏、下载、设置、登录）。
  - domain：业务模型与用例（播放状态、队列、收藏、搜索、歌词同步）。
  - data：API 客户端与本地缓存（封装 Functions 接口）。
- 平台差异：
  - AudioEngine 抽象：移动端 just_audio，桌面用对应适配实现（Web 暂不做）。
  - 下载与文件保存：平台适配层处理。

## 3. 功能映射（完整功能）
- 搜索与多数据源切换
- 播放队列管理（新增/删除/清空/导入/导出）
- 收藏列表（独立列表、批量操作、持久化）
- 播放模式（顺序/单曲/随机）
- 歌词同步与滚动高亮（手动滚动锁定）
- 背景取色与玻璃拟态主题（亮/暗模式）
- 多码率下载（128/192/320/FLAC）
- 探索雷达（自动补歌，分类可配置）
- 锁屏播放控制（移动端）
- 登录口令（密码页拦截）
- 调试控制台（快捷键/手势触发）

## 4. 数据流与存储
- Flutter → Cloudflare Functions → 返回搜索/播放/歌词/下载链接 → 播放器消费并更新状态。
- 登录口令：本地保存 token/session，后续请求自动携带。
- 播放/队列：内存为主，本地持久化。
- 收藏/历史：本地优先，保留后端扩展点。
- 缓存：搜索结果与歌词短期缓存。

## 5. 错误处理
- API 错误：统一 toast + 状态提示（网络异常/源不可用/解析失败）。
- 播放错误：自动重试一次，失败提供“切换源/重试”操作。
- 下载错误：提示原因与重试入口。

## 6. 测试策略
- 单元测试：播放队列、收藏逻辑、搜索响应解析。
- 集成测试：搜索→播放→歌词→收藏全流程。
- UI 回归：移动/桌面核心页面快照测试。

## 7. 后续扩展
- 自有后端替换：保持 ApiService 统一入口，替换 data 层实现。
- D1/账户体系扩展：可在登录/收藏与播放持久化上衔接。
